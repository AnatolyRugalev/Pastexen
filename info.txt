
=== Архитектура

1) Сервер, на основе пула потоков.

Принимает изображения и тексты, кладёт их на диск, возвращает ссылки:
Прямую ссылку на файл; ссылку на страницу с файлом (превью)

2) nginx - будет отдавать файлы + фронтенд

3) Скриншотилка - будет закачивать файлы

Базу данных на начальном этапе не используем.
У всех файлов - уникальное n-значное имя + корректное расширение.
По расширению определяется тип файла и срабатывает подсветка синтаксиса /
превью для картинок.

Протокол отправки файла клиентом на сервер:

Клиент отправляет:

proto=pastexen
version=1.0
type=jpg
size=142034

binary_data


Где jpg - тип файла; 142034 - размер файла в байтах; binary_data - содержимое файла

После получения файла сервер отвечает:

proto=pastexen
version=1.0
url=http://host.tst/file1.jpg
preview=http://host2.tst/file1-preview.php


url - прямая сылка на файл
preview - ссылка на превью


 === Дизайн

1) Главная страничка сервиса.
1 абзац текста - описание. Возможно несколько скриншотов приложения.
Крупная ссылка на скачивание. Чуть менее крупная ссылка на исходники.
Поискать похожие сайты скриншотилок, выделить как то наши преимущества
(pastebin, разные хоткеи, опенсорсность).

2) Страничка с превью скриншотом. Придумать, как будет выглядеть.

3) Страничка с исходным кодом с подсветкой синтаксиса.
Посмотреть пейстбин, гитхаб, сделать подобным образом.
